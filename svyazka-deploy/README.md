# Связка — processing ops

Приложение для процессинг-агента: дашборд, канвас связок, реестр, финансы, задачи, CRM и база знаний.
Собрано на React + Vite. Данные хранятся локально в браузере (`localStorage`).

---

## 1. Запустить у себя (локально)

Нужен установленный **Node.js 18+** (скачать: https://nodejs.org).

В терминале, находясь в папке проекта:

```bash
npm install      # один раз — поставит зависимости
npm run dev      # запустит локальный сервер
```

Открой адрес, который покажет терминал (обычно http://localhost:5173).
Это режим разработки — приложение работает, данные сохраняются в твоём браузере.

---

## 2. Собрать готовый сайт

```bash
npm run build
```

Создастся папка **`dist/`** — это и есть готовый статический сайт (HTML/CSS/JS).
Проверить сборку локально:

```bash
npm run preview
```

---

## 3. Выложить в интернет (выбери любой способ)

### Вариант А — Vercel (проще всего, бесплатно)
1. Зарегистрируйся на https://vercel.com (через GitHub).
2. Залей эту папку в репозиторий на GitHub.
3. В Vercel: **Add New → Project → Import** твой репозиторий.
4. Framework определится сам (Vite). Жми **Deploy**. Через минуту получишь ссылку.

### Вариант Б — Netlify Drop (без аккаунта и git, вручную)
1. Локально выполни `npm run build`.
2. Открой https://app.netlify.com/drop
3. Перетащи туда папку **`dist/`**. Сразу получишь публичную ссылку.

### Вариант В — любой хостинг
Залей содержимое папки `dist/` в корень своего хостинга/домена.

---

## Важно про данные

- Данные сохраняются в **браузере конкретного устройства** (`localStorage`).
  Открыл на другом компьютере/телефоне — там будет пусто.
- Чтобы работать **с разных устройств и в команде**, нужен серверный бэкенд
  (база данных + авторизация). Это следующий этап — текущая версия к нему готова:
  всё чтение/запись идёт через единый модуль `store` в `src/App.jsx`,
  его достаточно переключить на API.

## Структура

